DuckDuckGo - POST search feasibility test

This page submits the query as a real top-level POST navigation. If POST works, the address bar on the results page will show https://duckduckgo.com/ (or /html/) with no ?q=... after it.

A. POST to duckduckgo.com/ (main JS site)
B. POST to duckduckgo.com/html/ (no-JS lite version)
C. GET baseline (query will appear in URL - for comparison)

What to look for

  1. Submit form A. On the results page, check the address bar:
    • Pass: URL is https://duckduckgo.com/ with no query string, and results render.
    • Soft fail: URL becomes https://duckduckgo.com/?q=... - DDG redirected POST to GET.
  2. If A fails, try form B (the /html/ endpoint is the no-JS version - historically the most reliable POST target).
  3. Form C is the GET baseline - it should always put ?q=... in the URL.

DuckDuckGo's own settings page has a "Use POST method" option, but their endpoints accept POST regardless. If neither A nor B works, we'll inspect the form on duckduckgo.com directly with DevTools and replicate the exact payload.
